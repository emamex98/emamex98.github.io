TS Precompilation in Monorepos

First of all, let's not forget that whenever we write code in TypeScript, that is not exactly what gets executed at runtime in the browser. Instead, TypeScript code must first be compiled into browser-readable, good-ol' Javascript. With that in mind, TypeScript precompilation is the process of converting TypeScript code into JavaScript code before it gets executed.

Should you implement it in monorepos?

Now, when it comes to monorepos, there are a number of benefits of TypeScript precompiling. First, it can improve performance by reducing the amount of work that needs to be done at runtime, in addition to improving code quality by catching errors during compilation.

However, there are a few things to keep in mind when precompiling TypeScript code in monorepos:

  • Since monorepos often contain many packages accross the project, it is important to make sure that all of them are using the same version of TypeScript to avoid conflicts.
  • Also, it is important to configure the compiler to produce the correct output format to prevent issues from making it to producction, and this can be achieved by testing the code in several ways, such as unit, integration and acceptance tests.

Now again, no good thing can come with no downsides. Here are some of the downsides of precompiling TypeScript in morepos, and how these can be addressed:

  • TypeScript code can increase the build time of a monorepo. This is because the TypeScript compiler needs to be run on all of the TypeScript code in the monorepo, but this can be addressed by using a cache to store precompiled code and only precompile it again if changes are detected.
  • Precompiling TypeScript code can also increase the complexity of a monorepo. This is because it adds an additional step to the development process and requires developers to be familiar with the TypeScript compiler. However, using a text editor or IDE that supports TypeScript can help to make it easier to write TypeScript code and catch errors during development.
